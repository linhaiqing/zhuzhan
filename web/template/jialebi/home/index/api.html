API 使用说明 更新日期：2017年06月01日

申请API
 
GET 得到所有市场行情 (每10秒钟更新)
http://{$_SERVER['HTTP_HOST']}/api/trade/all/
 
GET 得到指定市场行情 (每10秒钟更新)
http://{$_SERVER['HTTP_HOST']}/api/trade/ticker?market={$config['mr_market']}
 
GET 得到全站市场深度 (每10秒钟更新)
http://{$_SERVER['HTTP_HOST']}/api/trade/depth?market={$config['mr_market']}
 
GET 得到全站历史成交 (每10秒钟更新)
http://{$_SERVER['HTTP_HOST']}/api/trade/trades?market={$config['mr_market']}
 
 
POST 得到自己账户余额

请求地址 ：http://{$_SERVER['HTTP_HOST']}/api/trade/balance

调用方式 ：POST

secret_key ：平台提供的32位私钥 (不用传参)


传入参数 ：

key，平台提供的32位公钥

nonce，时间戳，单位为秒

signature，使用md5(key&用户ID&nonce&secret_key)，顺序不可颠倒


返回： JSON格式

错误： {"err":'错误说明'}

成功： {"succ":'JSON格式的余额，含所有币种'}

 
 
POST 执行交易挂单操作

请求地址 ：http://{$_SERVER['HTTP_HOST']}/api/trade/submit_order

调用方式 ：POST

secret_key ：平台提供的32位私钥 (不用传参)


传入参数 ：

key，平台提供的32位公钥

nonce，时间戳，单位为秒

signature，使用md5(key&用户ID&nonce&secret_key)，顺序不可颠倒

type，1为买入挂单，2为卖出挂单，不可为空

market，交易市场

price，购买价格

amount，购买数量


返回： JSON格式

错误： {"err":'错误说明'}

成功： {"succ":'订单ID'}

 
 
POST 执行交易撤单操作

请求地址 ：http://{$_SERVER['HTTP_HOST']}/api/trade/cancel_order

调用方式 ：POST

secret_key ：平台提供的32位私钥 (不用传参)


传入参数 ：

key，平台提供的32位公钥

nonce，时间戳，单位为秒

signature，使用md5(key&用户ID&nonce&secret_key)，顺序不可颠倒

order_id，订单ID


返回： JSON格式

错误： {"err":'错误说明'}

成功： {"succ":'ok'}

 
 
POST 得到自己的挂单记录

请求地址 ：http://{$_SERVER['HTTP_HOST']}/api/trade/list_order

调用方式 ：POST

secret_key ：平台提供的32位私钥 (不用传参)


传入参数 ：

key，平台提供的32位公钥

nonce，时间戳，单位为秒

signature，使用md5(key&用户ID&nonce&secret_key)，顺序不可颠倒

market，交易市场 (不填则返回全部市场的挂单)

page，翻页，0为第1页，每页30条，不填则默认为第1页 排序ID大到小


返回： JSON格式

错误： {"err":'错误说明'}

成功： {"succ":'JSON格式的记录'}

解释： deal 已成交量 status 0交易中 1已完成 2已撤销
